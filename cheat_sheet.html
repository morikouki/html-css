23 メディアクエリ

メディアクエリ(@media)を利用すると、出力デバイスの特性や種類を判別し、条件を満たす場合にだけCSSの設定を適用することができる。

            ■画面幅が599px以下の際は「h1」のフォントサイズが「26px」になる

            @media (max-width: 599px) {
                .post h1 {
                    font-size: 26px;
                }
            }
        
            ■画面幅が600~768pxの間は、「h1」のフォントサイズが「32px」になる

            @media (min-width: 600px) and (max-width: 768px) {
                .post h1 {
                    font-size: 32px;
                }
            }
        
            ■画面幅が769px以上の際は「h1」のフォントサイズは「40px」になる

            @media (min-width: 769px) {
                .post h1 {
                    font-size: 40px;
                }
            }

メディアクエリは以下のような特性を判別することができる。

display: block / inline / run-in　フローレイアウト

displayプロパティの値を「block」「inline」「run-in」と指定すると、インナーディスプレイタイプ（レイアウトモード）を「フローレイアウト」にすることができる。

フローレイアウトでは、ボックスの中身がそれぞれのアウターディスプレイタイプ（ブロック / インライン / ランイン）の規則に従って並べられる。

テーブルの列の横幅の決定方法

            table-layout: 決定方法　

            テーブルの列の横幅の決定方法を指定することができる。

            全てのデータを読み込んでから決定する場合は「auto」を指定する

            １行目のデータを読み込んだ段階で蹴ってする場合は「fixed」を指定する
            「fixed」はテーブルの表示を速くする効果がある。
            tableタグの横幅をwidthプロパティで指定している場合のみ機能する

            指定できる値は「auto」と「fixed」だけ

            下記の例は
            tableタグに width: 600px; table-layout: auto;
            「プラン」に width: 80px;
            を指定している。

            「auto」を指定しているので、列の横幅は全てのデータを読み込んでから、同じ列に含まれる最も大きなセルの横幅に設定される。
            そのため、「プラン」に指定した、80pxではなく、２行目のセルの横幅で表示される。
